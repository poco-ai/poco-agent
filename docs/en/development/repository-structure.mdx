---
title: Repository Structure
description: High-level layout of the Poco codebase.
---

- `frontend/`: application routes, features, components, hooks, and shared utilities
- `backend/app/`: APIs, models, repositories, schemas, and services; also the source of truth for queued runs and session state
- `executor/app/`: execution engine, hooks, utils, and request schemas
- `executor_manager/app/`: polling/maintenance scheduler jobs, backend and executor integration clients, and the queue-driven runtime dispatch path (`RunPullService`)
